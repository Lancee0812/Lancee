## 2. 内容更新指南

### 2.1 更新个人信息
在 `index.html` 文件中找到 `<section id="about-me">` 部分，修改以下内容：
- 个人介绍文本
- 兴趣爱好列表
- 关于我段落内容

### 2.2 更新Fursona信息
在 `index.html` 文件中找到 `<section id="my-fursona">` 部分，修改以下内容：
- Fursona名称和介绍文本
- 特征描述（特别是关于角的描述）
- 背景故事文本

### 2.3 更新作品展示
在 `index.html` 文件中找到 `<section id="portfolio">` 部分，每个作品卡片的结构如下：

```html
<div class="portfolio-item" data-category="手作">
  <div class="portfolio-card">
    <div class="portfolio-image">
      <img src="https://picsum.photos/id/1/600/400" alt="作品名称" loading="lazy">
    </div>
    <div class="portfolio-content">
      <h3>作品名称</h3>
      <p>作品描述内容...</p>
      <div class="portfolio-tags">
        <span class="tag">手作</span>
      </div>
    </div>
  </div>
</div>
```

要添加新作品，请复制上述结构并粘贴在现有作品卡片之间，确保：
1. 更新图片URL（可使用picsum.photos或替换为自己的图片）
2. 修改作品名称、描述和标签
3. 确保 `data-category` 属性与标签内容一致

### 2.4 更新联系方式
在 `index.html` 文件中找到 `<section id="contact">` 部分，修改社交媒体链接。每个社交平台链接的结构如下：

```html
<a href="#" class="social-link" aria-label="访问我的QQ空间">
  <i class="social-icon">QQ</i>
</a>
```

只需更新 `href` 属性为实际的社交平台链接。

## 3. 样式调整指南

### 3.1 修改颜色方案
在 `styles.css` 文件顶部的 `:root` 选择器中，可以修改以下CSS变量：

```css
:root {
  /* 主色调 */
  --color-white: #FFFFFF;
  --color-light-gray: #F5F5F7;
  --color-medium-gray: #86868B;
  --color-dark-gray: #1D1D1F;
  --color-black: #000000;
  
  /* 强调色 */
  --color-accent-1: #3A7CA5;  /* 低饱和水蓝色 */
  --color-accent-2: #2A9D8F;  /* 高饱和青绿色 */
  
  /* 支持色 */
  --color-success: #34C759;
  --color-warning: #FF9500;
  --color-error: #FF3B30;
}
```

### 3.2 调整字体大小和间距
可以修改 `styles.css` 文件中的字体大小和间距变量：

```css
/* 字体大小 */
:root {
  --font-size-base: 16px;
  --font-size-small: clamp(0.875rem, 1vw, 1rem);
  --font-size-large: clamp(1.25rem, 2vw, 1.75rem);
  --font-size-xlarge: clamp(1.75rem, 3vw, 2.5rem);
  --font-size-xxlarge: clamp(2.5rem, 5vw, 4rem);
}

/* 间距单位 */
:root {
  --spacing-unit: 4px;
  --spacing-sm: calc(var(--spacing-unit) * 2);  /* 8px */
  --spacing-md: calc(var(--spacing-unit) * 4);  /* 16px */
  --spacing-lg: calc(var(--spacing-unit) * 6);  /* 24px */
  --spacing-xl: calc(var(--spacing-unit) * 8);  /* 32px */
  --spacing-xxl: calc(var(--spacing-unit) * 12); /* 48px */
  --spacing-xxxl: calc(var(--spacing-unit) * 24); /* 96px */
}
```

### 4.2 图片替换最佳实践
- 准备的图片应该有合适的尺寸（推荐：缩略图600x400像素）
- 使用Web友好格式（JPEG、PNG或WebP）
- 压缩图片以优化加载速度
- 替换时保持文件名一致性或更新HTML中的引用路径

## 5. 常见问题排查

### 5.1 图片不显示
- 检查图片URL是否正确
- 确保图片路径大小写与实际文件名一致
- 验证图片文件是否存在于指定位置

### 5.2 交互功能不工作
- 检查 `scripts.js` 文件是否正确加载（在浏览器控制台中查看是否有错误）
- 确认HTML元素的ID或class名称与JavaScript中的选择器匹配
- 验证事件监听器是否正确绑定

### 5.3 响应式布局问题
- 对于移动设备上的显示问题，检查媒体查询断点：
  - 移动设备：max-width: 768px
  - 平板设备：max-width: 1024px
- 确保使用的是相对单位（rem, em, %, vh, vw）而非固定像素值

## 6. 性能优化建议

### 6.1 图片优化
- 为所有图片添加 `loading="lazy"` 属性（已在当前代码中实现）
- 使用适当的图片尺寸，避免加载过大的图片
- 考虑使用响应式图片标签：

```html
<picture>
  <source media="(max-width: 768px)" srcset="small-image.jpg">
  <img src="large-image.jpg" alt="描述" loading="lazy">
</picture>
```

### 6.2 代码优化
- 合并和压缩CSS和JavaScript文件（如果网站流量较大）
- 移除未使用的CSS和JavaScript代码
- 减少DOM操作次数以提高交互性能
